# Dataverity

Dataverity — это сервис для мониторинга качества данных, проверки датасетов и обеспечения воспроизводимости data-pipeline.

Цель проекта — помочь организациям доверять своим данным, обнаруживая аномалии, проверяя бизнес-правила и отслеживая происхождение данных.

---

# Что такое Dataverity

Современные организации принимают решения на основе данных, однако data-pipeline часто сталкиваются с проблемами:

* дубликаты записей
* пропущенные значения
* ошибки схемы данных
* ошибки трансформации
* незафиксированные изменения данных
* невозможность воспроизвести расчёты

Dataverity предоставляет централизованный сервис для мониторинга качества данных на всех этапах их обработки.

---

# Основные возможности

## Мониторинг качества данных

Система оценивает датасеты по нескольким показателям:

* поиск дубликатов
* доля пропущенных значений
* проверка схемы данных
* обнаружение статистических аномалий
* проверка бизнес-правил

Все показатели объединяются в **Data Quality Score**.

---

## Правила проверки данных

Пользователь может задавать правила:

* уникальность колонок
* диапазоны значений
* допустимые категории
* ограничения схемы
* пользовательские бизнес-правила

Нарушения автоматически фиксируются и сохраняются в системе.

---

## Воспроизводимость пайплайнов

Каждый запуск пайплайна сохраняет:

* версию датасета
* business date
* этапы трансформации
* конфигурацию пайплайна
* правила проверки

Это позволяет точно воспроизводить расчёты.

---

## Lineage данных

Dataverity отслеживает происхождение данных.

Граф lineage показывает:

* источники данных
* трансформации
* промежуточные датасеты
* зависимые наборы данных

---

## Наблюдаемость данных

Система отслеживает:

* запуски пайплайнов
* метрики качества данных
* аномалии
* изменения схемы
* нарушения правил

# Подготовка финансовых данных

Dataverity может использоваться для подготовки данных, необходимых для расчёта финансовых нормативов кредитно-финансовых организаций.

Сервис обеспечивает:

* проверку качества данных
* согласованность информации
* воспроизводимость расчётов
* прозрачность происхождения данных

Это особенно важно при подготовке данных для расчёта показателей финансовой устойчивости.

# Запуск проекта

Клонировать репозиторий:

```
git clone https://github.com/Hanemiyaaaa/DataVerity
```

Запустить сервисы:

```
docker compose up
```

Доступные сервисы:

```
Frontend: http://localhost:3000
Backend API: http://localhost:8000
Airflow: http://localhost:8080
```
